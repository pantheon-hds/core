Verification Flow
Core principles

Anonymous submission — no name, no avatar, no history shown to judges
Random judge selection — Gold+ players of the same game
Blind independent voting — judges cannot see each other's votes
One appeal allowed — new set of random judges


Phase 1 — 0 to 200 players

Player completes challenge and records full uncut video
Uploads to YouTube as unlisted link
Submits link to platform with challenge name
Founder reviews manually and decides

Video requirements:

No cuts
HUD visible at all times
End-game stats shown


Phase 2 — 200+ players

Player submits video (same requirements as Phase 1)
System strips identity — name, avatar, and history hidden
5 random Gold+ judges from the same game are selected
Each judge watches the video and votes independently
Judges cannot see each other's votes until all 5 have voted
3 out of 5 votes needed to pass
Identity revealed only after final decision


Appeal process

Available only on rejection
One appeal per submission
New 5 random judges selected — completely different pool
Same blind anonymous process
Final decision is binding


Anti-cheat checklist (manual, early stage)

 Video is uncut and continuous
 HUD is visible throughout
 End stats are shown
 No suspicious edits or frame skips
 Challenge conditions clearly met


Future automation (roadmap)

Steam API integration for Gold rank (automatic)
Save file parsing for Platinum verification
Steam Stats API for challenge conditions
Developer partnership for native verification tokens
